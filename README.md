# Printer Asset Manager

Printer Asset Manager is a full-stack web application for managing industrial printers, backup printers, printer health checks, preventive maintenance, issues, spare parts, cartridges, label recipes, and support knowledge.

It includes:
- Role-based login and approval workflow
- Printer inventory, health, and issue tracking
- Spare parts management and cartridge monitoring
- Application support and technical support workflows

## Quick start for a new machine

1. Install prerequisites:
   - Node.js 18+ (includes `npm`)
   - PostgreSQL
2. Clone the repository:
   ```bash
git clone <repo-url> PrinterAssetWeb
cd PrinterAssetWeb
```
3. Install dependencies:
   ```bash
npm run install:all
```
4. Configure backend environment:
   - Create `backend/.env`
   - Add database and JWT settings
5. Initialize PostgreSQL database
6. Create the admin account
7. Start the backend and frontend
8. Login with the admin credentials below

## Install dependencies

From the repository root:

```bash
npm run install:all
```

Or install separately:

```bash
cd backend
npm install
cd ../frontend
npm install
```

## Backend configuration

Create a file `backend/.env` with values similar to:

```env
PORT=5000
DATABASE_URL=postgresql://postgres:your_password@localhost:5432/printer_ms
DB_SSL=false
JWT_SECRET=change-this-secret
EMAIL_USER=your.gmail.address@gmail.com
EMAIL_PASSWORD=your_gmail_app_password
PRINTER_MONITOR_INTERVAL_MS=60000
PRINTER_MONITOR_CONCURRENCY=6
```

> If your backend reads separate DB fields instead of `DATABASE_URL`, set those accordingly.

## Database setup

Initialize PostgreSQL and apply schema:

```bash
createdb -U postgres printer_ms
psql -U postgres -d printer_ms -f backend/schema.sql
```

Alternatively, use the backend setup script:

```bash
cd backend
node setup-db.js
```

## Create or reset the admin account

Run this after the database is ready:

```bash
cd backend
node recreate-admin.js
```

This creates or recreates the admin user with support access.

## Run backend

Start the backend in development mode:

```bash
npm run dev:backend
```

Or directly:

```bash
cd backend
npm start
```

## Run frontend

Start the frontend app:

```bash
npm run dev:frontend
```

Open the Vite URL shown in the terminal, usually `http://localhost:5173`.

## Troubleshooting

- If backend fails to connect to PostgreSQL, verify `backend/.env` and database credentials.
- Ensure PostgreSQL is running and the `printer_ms` database exists.
- If login does not work, run `node backend/recreate-admin.js` again after the database is ready.

## Notes

- `frontend/dist` is generated by Vite and should not be edited manually.
- The backend uses JWT authentication and PostgreSQL for user data.
- The admin login above is the default admin for new machine setup.
